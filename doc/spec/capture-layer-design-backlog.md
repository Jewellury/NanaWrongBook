# 需求备忘 · 采集层交互设计（未来轮次）

> 来源：`doc/research/得到大脑捕获交互调研报告.md`。触发自第 0 阶段人肉回路的录音采集需求。
> 性质：未来"采集/归因轮"或"session UI 轮"的**设计输入**。现在不建。
> 但其中几条**可立即指导人肉回路怎么跑**（零成本，见文末）。
> 守：P1（原音是真相、转写是索引）/ P4（不评判）/ P5（周中无手机、周末采集）。

## 五条值得借的交互模式（报告提炼，我背书）
1. **场景入口替代空白页**——任务导向 IA，降低青少年"该点哪"的空白焦虑。我们的按钮应是"拍一下这道题""补一段你当时怎么想的""把这周卡住的都留下来""回放上次讨论"。
2. **多模态围绕同一条 artifact 生长**——题图 + 原音 + 舅舅追问 + AI 提要 = 一件 case 的不同层，不是四个孤岛。
3. **原文版 / AI优化版并存，AI 不覆盖原始**——直接落 P1。任何 AI 提要都能点回音频时间轴。
4. **AI 总结录后自动浮出、不抢正文位**——采集阶段别打断，整理阶段 AI 才发光（AI 放第二拍）。
5. **自动沉淀到"可回找时间线"**——但改教育语义命名："这周讨论""上次讲到哪""卡过的地方"，不要"记忆资产"这种抽象词。

## 我们的核心超越点（报告指出 Get 未证实做到）
**题图固定可见的陪伴式录音**：进入"补口述"后，上半屏永远是题图、下半屏录音波形 + 实时转写。孩子口述数学卡点时要反复看题面/图形/自己的笔迹。这是我们该做对、而 Get 没证实做到的地方。

## case 对象模型（报告的"错题卡片盒"）
每张卡四层：**题图 / 原音 / 逐字稿 / AI 提要**。
- 给舅舅第一眼：提要 + 关键时间点（可点回音频）；
- 给孩子第一眼：题图 + "上次你自己怎么说的"（不是一坨模型总结）。

## 后端借飞书妙记 / 通义听悟
Get 是采集前端；飞书妙记/通义听悟强在**录后工作台**（音文对照回看、倍速、跳空白、说话人筛选、快速回找）。素材攒起来后，真正值钱的不是更花的 AI，是"能不能快速回听/回看/回找那句原话"。将来整理后端借它们的手感。

## 归属与时机
- 归属：未来"采集/归因轮"（接火山方舟 ASR 时）或"session UI 轮"。到那轮 `/plan` 时吸收本备忘。
- 时机：**等真实使用攒够再做。现在不建。**

## ★ 可立即用于人肉回路的（零成本，本周就能用）
即便不写一行代码，这几条今天就能指导舅舅怎么带她跑回路：
- 周末**批量拍 5–10 题先不解释**，再逐条补口述（别逼她拍一题讲一题——报告说得对，这更现实）；
- 口述时**题纸放在眼前**（纸质就够，正好是我们的"题图固定可见"）；
- **留原始录音**（手机语音备忘录），别只留转写/记忆——原音是真相源（P1）；
- 舅舅**异步回听**再追问，不评判（"只听和引导"，P4）。
