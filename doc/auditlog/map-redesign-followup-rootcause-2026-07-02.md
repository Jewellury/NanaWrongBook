## 问题 1a：图谱文字重叠（"可以先看"与节点名叠在一起）

### 症状
图谱里 frontier 节点上方显示"可以先看"文字、下方显示节点名，与灰底图的 cluster labels 和灰节点名三层叠在一起，在手机 368px 画布上不可读。

### 根因（Phase 1 确认）
`knowledge-map-canvas.tsx:311-320`：每个 frontier 节点渲染两个 `<text>`：
- L311-320：`{nextLabel}`（"可以先看"/"下一个"），位于节点**上方** `y + LABEL_Y_OFFSET`（-14）
- L330-339：`{node.name}`，位于节点**下方** `y + TEXT_Y_OFFSET`（16）

叠加灰底图（L220-226 的 CLUSTER_LABELS + 每个节点的灰名），三层文字在密集布局里互相覆盖。

### 修复方案（用户建议正确）
**去掉 frontier 节点上的 `{nextLabel}` 文字标签**（L311-320 整个 `<text>` 块删除）。用蓝色虚线圆圈本身的颜色/形态区分"frontier"——蓝色虚线圈已经是足够的视觉信号，不需要文字标签重复。图例里已有"可以先看/下一个"蓝色样块说明含义。

节点名（L330-339）保留——那是节点的标识，不是状态标签。

## 问题 1b：列表视图点详情卡无法收回

### 症状
切换到列表模式，点击任一节点（如"可以先看"组里的），弹出详情卡，但无法关闭收回。

### 根因（Phase 1 确认）
`knowledge-detail-card.tsx:66`：容器 `fixed bottom-0 left-0 right-0 z-50 flex items-end justify-center`。
遮罩 `absolute inset-0 bg-black/10 onClick={onClose}`（L68-72）。

**根因 A：遮罩透明度太低**（`bg-black/10` = 10% 黑色），在手机上几乎不可见，用户不知道遮罩存在、不知道点哪里关闭。

**根因 B：任务 D 删了"关闭"按钮**（DP4 restyle），唯一的关闭方式是点遮罩——但遮罩不可见，等于没有可发现的关闭途径。

**根因 C（z-index 可能冲突）**：详情卡 `z-50`，但列表视图本身在 `flex-1` 容器里，如果列表内容有 `z-30` 以上的元素（如浮层入口按钮 `z-30`），可能拦截了遮罩的点击事件。不过详情卡用 `fixed` 定位脱离了正常文档流，z-50 应该够高。

### 修复方案
**最小修复**：遮罩透明度从 `bg-black/10` 提高到 `bg-black/30`（可见但不刺眼）；**加回一个明确的关闭方式**——不恢复大按钮，而是在卡片右上角加一个小 `×` 图标（lucide `X`），或恢复 grip 条的可点击关闭（点 grip 条关闭）。推荐：遮罩加深到 30% + grip 条加 `onClick={onClose}` + `cursor:pointer`（grip 本来就是设计稿里暗示"下拉关闭"的交互元素，点击关闭最自然）。

## 问题 2：首页"周末小检查"与 RecapBar 重复 + "点了没反应"

### 症状
首页 ActionCard "周末小检查" → `/nana/session`；下方 RecapBar 链接"去做小检查，点亮它们 →" 也 → `/nana/session`。两个入口跳同一个页面，重复。且"点了没反应"。

### 根因（Phase 1 确认）
**重复确认**：`page.tsx:117` ActionCard href="/nana/session"；`recap-bar.tsx:52` Link href="/nana/session"。**确实重复**——本轮 DP3 改 RecapBar 链接从 `/nana/knowledge-map` 改到 `/nana/session`，结果跟 ActionCard 撞了。

**"点了没反应"根因**：`/nana/session` 页面加载后，如果该用户没有 session 数据，会显示空状态。用户点了链接，页面跳转了，但 session 页面内容为空（或加载慢），用户感知为"没反应"。这不是链接本身的问题，而是**目标页面在空状态下的 UX 不够明显**。当然，也可能是 RecapBar 的 Link 被其他元素遮挡（z-index 问题），但更可能是前者。

### 修复方案
**去重**：RecapBar 去掉 `/nana/session` 链接，改成**纯提示文案**（不带 Link）。RecapBar 的价值是"给你一个状态回顾"（收过题/点亮了），不是"另一个入口"。行动入口由三张 ActionCard 承担，RecapBar 只做状态反馈。

具体：两个分支都去掉 `<Link>`，文案改为纯文字提示：
- 只 collected 分支："你最近收过题的知识点有 N 个。还没做小检查，做完就能点亮它们 ✦"（无链接）
- 有点亮分支："上次你点亮了：X。你的地图上已经有 N 个光点了 ✦"（无链接）
