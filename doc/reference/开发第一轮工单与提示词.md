## 提示词 A：开发第一轮（知识图谱数据层）

```
你是这个项目的开发者。项目基于开源的 wrong-notebook（Next.js 16 App Router / React 19 /
SQLite / Prisma / NextAuth / PWA），我要在它之上增量开发一个"个性化数学诊断辅导系统"。
本轮只做第一个增量：知识图谱数据层。严格遵守以下约束。

【最高原则】
1. 绝不修改 wrong-notebook 已有的 Prisma 模型和已有表结构。所有新功能通过【新增表】实现，
   与既有表通过新增的关联表挂接。目的是日后能 rebase 上游更新而不冲突。
2. 不写 UI、不写诊断算法、不调用任何 AI。本轮交付物是：迁移文件 + 种子导入脚本 + 一个
   内存图谱模块 + 单元测试。能 `npm run seed` 把种子灌进库、能跑通图遍历测试，即为完成。
3. 所有代码注释和 commit 用中文。

【任务1：Prisma 增量 schema】
在现有 schema.prisma 末尾追加以下模型（不要改动已有部分）。字段含义见注释：

model KnowledgeNode {
  id            String   @id            // "BG001" / "M1-11" / "M2a-03"
  name          String
  layer         String                  // foundation | mainline
  stage         String                  // 学段，如 "七上" "必修一3.1"
  judgeCriteria String
  sampleItem    String?
  teachingNotes String?                 // 迁移类比/通法步骤/易错点
  tier          String?                 // A | B | C | D | null（保底层级，调度主键）
  videoLinks    Json?
  edgesOut      KnowledgeEdge[] @relation("src")
  edgesIn       KnowledgeEdge[] @relation("tgt")
  mainlines     NodeMainline[]
  states        StudentNodeState[]
}
model KnowledgeEdge {
  sourceId String
  targetId String
  type     String                       // prerequisite | tool
  source   KnowledgeNode @relation("src", fields:[sourceId], references:[id])
  target   KnowledgeNode @relation("tgt", fields:[targetId], references:[id])
  @@id([sourceId, targetId])
}
model Mainline {
  id       String @id                   // "M1".."M8"/"M2a"/"M2b"/"M0"
  name     String
  priority Int
  weight   Float?                        // 高考严格年均分值，供 ROI 排序
  nodes    NodeMainline[]
}
model NodeMainline {
  nodeId String
  mainlineId String
  node KnowledgeNode @relation(fields:[nodeId], references:[id])
  mainline Mainline @relation(fields:[mainlineId], references:[id])
  @@id([nodeId, mainlineId])
}
model StudentNodeState {
  studentId String
  nodeId String
  node KnowledgeNode @relation(fields:[nodeId], references:[id])
  masteryProb Float @default(0.0)
  status String @default("untested")    // stable|uncertain|gap|untested
  slipFlag Boolean @default(false)
  lastEvidence DateTime?
  @@id([studentId, nodeId])
}
model Misconception {                    // 误解库（错题库调研的四联体）
  id          String @id @default(cuid())
  board       String                     // 板块
  errorType   String                     // 一级标签:概念性|程序性|计算性|符号表示|条件前提缺口|完备性缺口
  crossTag    String?                    // 跨章复用标签:定义域优先|等价变形守恒|完备分类|取等可达|对象类型意识 等
  manifestation String                   // 错误的具体表现（最小示例）
  misbelief   String                     // 背后的概念误解
  rootNodeId  String?                     // 暴露的真正知识缺口 → KnowledgeNode.id
  probeCue    String                      // 教师追问/反例
  evidence    String                      // 证据等级 A|B|C
}
model MistakeNode {                       // 错题↔节点关联（挂接 wrong-notebook 既有 Mistake 表）
  mistakeId String                        // → 既有 Mistake.id（不改其表）
  nodeId    String
  confirmed Boolean @default(false)       // VLM打标后是否人工确认
  @@id([mistakeId, nodeId])
}

生成迁移：npx prisma migrate dev --name add_knowledge_graph

【任务2：种子导入脚本】
我会提供一个种子数据文件 seed_graph_batch1.ts（结构见我贴的内容）。写一个 prisma/seed_graph.ts：
- 读取 mainlines / foundationExtra / M1nodes / M2aNodes / bridges / videoMapSample 等导出对象
- 幂等导入（用 upsert，可重复跑不报错）
- foundationPrereq 里目前是文字描述（如 "BG101解不等式"），本轮先原样存进 teachingNotes 的
  一个子字段或忽略；真正的边只导入 bridges 和 samePrereq 里已经是 ID 的部分
- samePrereq 数组里的每一项，对应建一条 KnowledgeEdge(type=prerequisite)
- bridges 数组里 src/tgt 是主线级，本轮先存成"主线→主线"的占位边（节点级细化以后再做），
  或单独存一张 MainlineBridge 表，你决定哪种更干净
- 在 package.json 加 "seed": "tsx prisma/seed_graph.ts"

【任务3：内存图谱模块 lib/graph.ts】
应用启动时把全图加载进内存（节点数预计 <400，边 <1000），导出单例，提供：
- prereqsOf(nodeId): 直接前置节点列表
- allPrereqsOf(nodeId): 递归全部前置（用于"答对则祖先链全 stable"传播）
- dependentsOf(nodeId): 直接依赖它的节点
- mainlineSubgraph(mainlineId): 该主线及其前置涉及的节点子图
- 注意防环：图理论上是 DAG，但加载时要检测环并报警（数据错误早发现）

【任务4：单元测试】
用项目已有的测试框架（没有就用 vitest）写测试：
- 种子导入后节点数、边数符合预期
- allPrereqsOf("M2a-03") 应包含其声明的前置链
- 图无环检测通过
- 随机抽 3 个节点，其 prereqsOf 结果与种子文件声明一致

【交付】
分成清晰的 commit：①schema迁移 ②种子脚本 ③graph模块 ④测试。
每个 commit 能独立通过。完成后告诉我怎么验证（跑哪些命令、预期输出）。
遇到 wrong-notebook 既有代码与我的要求冲突的地方，停下来问我，不要擅自改既有表。
```

## 提示词 B：真题逐题解析（给负责解析的外部 AI）

```
我有新高考全国I卷数学真题的整页图片（命名 page-01.jpg ... page-NN.jpg，由真题解析版docx
转PDF再按150DPI导出，公式清晰可读）。请你逐页看图，把整张试卷完整、忠实地转写成结构化文本。

【重要】文档里的公式、根号、分式、几何图形原本都是图片，所以你必须【看图转写】，
不要假设有现成文本可抓。每道题都要转写完整，一题都不能漏。

每道题输出以下字段（YAML 或 Markdown 表，逐题一条）：
- year: 年份
- number: 题号
- type: 单选/多选/填空/解答
- score: 分值
- stem: 完整题面，数学符号一律用 LaTeX 写（如 $A=\{x\mid -5<x^3<5\}$），几何图形用文字描述其要素
- options: 选择题的四个选项（LaTeX），非选择题留空
- answer: 答案
- analysis_brief: 解析要点（基础题和大题第一问要写出标准解法步骤；压轴题只写思路一两句即可）
- has_figure: 该题是否含几何图形（true/false）

【转写纪律】
1. 数字、正负号、上下标、根号范围、集合区间开闭，是最容易看错的地方，请逐字核对原图。
2. 看不清或不确定的地方，不要猜，在该字段后标注 [存疑] 并说明哪里看不清。
3. 几何图（立体几何、解析几何配图）你无法用LaTeX表达的，用文字尽量描述清楚已知条件，
   并标注 has_figure: true，提示需要人工补图。
4. 不要改写、不要简化题目，保持原题措辞。

先做完一整年（如2024）的全部19题，再做下一年。每年做完停一下，等我确认格式再继续。
```

这份产出回来后，由我（复核方）逐题核对，重点查 LaTeX 转写的数字/符号/区间，确认后整理成可入库的真题种子库。

## 配套：误解库种子（misconceptions 表，取自错题库调研）

错题库调研的 20+ 条误解四联体，可直接整理成 Misconception 表的种子。映射规则：rootNodeId 指向
已有节点 ID（如 "log(a+b) 误拆" → M2a-38；"分段函数单调性" → M2a-13 附近；"复数当向量写" →
M1-26）。这部分我下一批连同 M2a 节点补全一起产出结构化种子文件。

跨章复用标签（crossTag）首批取这五个，它们会横跨多个节点反复触发，是诊断引擎抓"系统性缺口"
的关键：定义域优先 / 等价变形守恒 / 完备分类讨论 / 取等可达性 / 对象类型意识。
```
