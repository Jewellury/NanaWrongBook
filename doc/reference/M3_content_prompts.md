# M3 内容生产提示词（两份）

> 用途：给外部/调研 AI 的两份工单。
> A = 真题逐题解析（修订版，替代旧工单 Prompt B）；B = 初诊弹药（配题）生成。
> 铁律：两份产出回来都由我方**逐题人工复核**——重点查**数学答案正确性 + 数字/符号/区间开闭**。AI 算错、瞎编映射是高频病，一律当线索不当结论。

---

## 提示词 A：真题逐题解析（修订版）

> 相比旧版的两处改动：①明确这是**第一阶段·忠实转写**，不做节点映射（映射是第二阶段教研活，转写 AI 不懂我们的编码本，硬标必污染图谱）；②每题加一个**白话"考点关键词"**字段，只为后面人工映射省力，**不映射到我们的节点编码**。

```
我有新高考全国 I 卷数学真题的整页图片（page-01.jpg ... page-NN.jpg，由真题解析版 docx
转 PDF 再按 150DPI 导出，公式清晰可读）。请逐页看图，把整张试卷完整、忠实地转写成结构化文本。

【这是第一阶段：忠实转写】
你的唯一任务是"把图上的题一字不差变成文本"。不要判断这题考哪个知识点编码、不要分难度等级、
不要分类——那是后面单独一步的事。你只管转写准确、完整。

【重要】文档里的公式、根号、分式、几何图形原本都是图片，必须【看图转写】，
不要假设有现成文本可抓。每道题都要转写完整，一题都不能漏。

每道题输出以下字段（YAML 或 Markdown 表，逐题一条）：
- qid: 年份-题号（如 2024-T3，做来源追溯用）
- year: 年份
- number: 题号
- type: 单选/多选/填空/解答
- score: 分值
- stem: 完整题面，数学符号一律 LaTeX（如 $A=\{x\mid -5<x^3<5\}$），几何图形用文字描述要素
- options: 选择题四个选项（LaTeX），非选择题留空
- answer: 答案
- analysis_brief: 解析要点（基础题和大题第一问写出标准解法步骤；压轴题只写思路一两句）
- has_figure: 是否含几何图形（true/false）
- 考点关键词: 用大白话写这题大致考什么（2-5 个词，如"集合交集、区间开闭"或"函数单调性、定义域"）。
  这只是给人看的线索，不要套任何编码体系。

【转写纪律】
1. 数字、正负号、上下标、根号范围、集合区间开闭，最容易看错，逐字核对原图。
2. 看不清/不确定的地方不要猜，在该字段后标 [存疑] 并说明哪里看不清。
3. 几何图无法用 LaTeX 表达的，用文字尽量描述清楚已知条件，标 has_figure: true 提示人工补图。
4. 不改写、不简化、不"优化"题目，保持原题措辞。

先做完一整年（如 2024）全部题目，再做下一年。每年做完停一下，等我确认格式再继续。
```

**第二阶段（不在本提示词内，我方/专门提示词做）**：拿转写结果 + 我们的节点清单，逐题标
`nodeId` + `tier` + `role`。绝不让转写 AI 代劳。

## 提示词 B：初诊弹药（配题）生成

> 目标：为**已在种子库中、tier=A 的微节点**（先做 M1 的 A 层 + 地基 BG100-104 + M2a 的 A 层）
> 生成诊断用题，入 `Item` 表（nodeId + role + stem + answer）。
> 关键认知：这些是**地板探针题**，不是考试题——真题喂不出这一层（真题水位太高、且常一题多考点，做不了干净探针）。
> 用法：把下面提示词 + 一批节点数据（从 seed_graph_batch1.ts 取 id/name/judgeCriteria/sampleItem/tier/stage）一起发给 AI。

```
【角色与场景】
你在为一个面向"数学基础非常薄弱的高三学生"的诊断系统生产【初诊题】。
这个学生数学很弱（150 分卷曾只考 30-40 分），目标是从地基摸起、保底提分。
你产的不是考试题，是【地板题/探针题】——用来摸清"她到底会不会某一个最小知识点"。

【五条铁律】
1. **一题只测一个节点**：每道题只对应我给你的某一个节点的 judgeCriteria（判定标准）。
   绝不把多个知识点揉进一题——否则她做错了，我们无法判断是哪个点崩的。
2. **地板难度**：面向能力下限。boundary 题尽量一步到位、不绕弯、数字干净，能让"刚会"的人做对、
   "不会"的人卡住。不要为了显得有水平而加难度。
3. **只对给定节点产题**：我会给你一批节点，每个含 id / name / judgeCriteria / sampleItem / tier / stage。
   不许发明新节点、不许改 id、不许给我没给的节点产题。
4. **答案必须对**：数学符号一律 LaTeX；每题必须给答案且必须正确。
   你若对答案没十足把握，标 [存疑] 并说明，**绝不硬编一个看起来对的答案**。
5. **不要照抄 sampleItem**：以它为锚生成新题（sampleItem 将来可能就是其中一道 boundary）。

【每个节点产出哪些题（按 role）】
- boundary ×1（必产）：判断"会不会"的最小题，直接对应 judgeCriteria。这是初诊主力。
- variant ×1（必产）：与该节点 boundary【同构】——同方法、同步骤，只换数字/情境，
  用于"隔天复测"验证她是真会还是蒙对。必须注明它复测的是哪道 boundary。
- concept ×1（节点偏概念理解时才产）：一道能暴露"答案对、其实没懂"的题，
  或附一句费曼式追问（让她讲"为什么这步能这么做"）。
- drill ×1~2（节点偏计算/程序熟练时才产）：同类反复练手的题，练运算自动化。

（可选）boundary 若是选择题，四个干扰项尽量对应"常见错误"，并简注每个错误选项踩的是什么坑；
拿不准就跳过，不要硬凑。

【输出格式】逐题一条（YAML 或表）：
- nodeId
- role: boundary | concept | variant | drill
- stem（LaTeX）
- answer
- pairWith（variant 专用：填它复测的 boundary 在本节点下的序号/题面摘要）
- conceptCue（concept 专用：一句费曼追问）
- distractorNote（可选：干扰项对应的误解）
- note（存疑或备注）

【节奏】先做我这一批给的节点（M1 的 A 层），做完【停下来】，
等我确认题目数学正确性和格式没问题，再做下一批。不要一次性铺开所有节点。
```

**首批范围**：种子库里已有、tier=A 的节点——M1 的 A 层（M1-04~M1-17、M1-24~M1-30 等）+
BG100-104 + M2a 的 A 层（M2a-01/03/04/09/17/32/38/42 等）。
**地基 BG001-099 的题先不做**——那些节点还没进图谱，产了也是孤儿题，等后续批次种子导入后再配。

## 我方复核清单（两份产出回来都走这个）

| 检查 | 重点 |
|------|------|
| 数学正确性 | **逐题验答案**——AI 算错是高频病，尤其根号/区间/正负号 |
| 单点隔离（仅 B） | 每题确实只测一个节点，没有偷偷掺别的考点 |
| 难度（仅 B） | boundary 是不是真"地板"，没拔高 |
| 转写忠实（仅 A） | 数字/符号/区间开闭逐字对原图；[存疑] 项人工补 |
| 不信 AI 给的归类 | A 的"考点关键词"、B 的 distractorNote 都当线索，映射/采纳由我方定 |

复核通过后，整理成 `Item` 表种子文件（B）/真题种子文件（A），走三代理 `/plan → /execute → /audit` 入库。
