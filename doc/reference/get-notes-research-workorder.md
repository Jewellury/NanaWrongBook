# 工单 · 调研 Get笔记（得到大脑）错题录音整理流程

> 触发：用户笔记《我如何使用AI系列：AI协助错题整理的方法》（骑蜗牛的小书童, 2026-06）。
> 可借鉴核心 = 数学错题"**拍照 + 追加录音笔记 → 自动转写提要点 → 知识库 → 举一反三**"。

## 定调（给调研 AI，先读这段再查）
这篇文章的方法论**几乎完整印证了我们已有路线**：录音讨论=我们的"重做+录音/Newman追问"；喂教材定位知识点=我们的**知识图谱**（我们已是结构化、带前置依赖、持久的版本，比它临时喂教材更强）；从错题看概念缺口=我们的诊断引擎。
**所以本次调研只聚焦"采集体验"那一段（拍 + 录音转写提要点），不要去研究它的出题/知识库——那些我们自己做得更结构化。**

## 调研对象
**Get笔记 / 得到大脑**（先确认出品方与现名）。重点功能：拍照建笔记、追加**录音笔记**、自动转录、提取要点/摘要、知识库、AI 错因分析与举一反三。

## 必答问题
1. **录音转写质量**：中文口语 + 数学术语（分式/根号/"约分""通分""进位"）转得准不准？对数学讨论场景够用吗？
2. **能否导出 / 有无 API**：转写稿和要点能否导出（md/txt）？有无开放接口？—— 决定它能否给我们当**素材源**（喂 Newman prompt 调试 + 黄金测试集）。
3. **采集摩擦**：拍+录音+提要点整套几步？适不适合一个高三孩子 + 远程舅舅异步用？
4. **隐私 / 数据**：她的录音和错题存哪、能否删除、隐私条款。（我们对她的数据安全很在意）
5. **免费额度 / 价格**。
6. **横向对比一句**：通义听悟 / 飞书妙记 / 讯飞听见 等，在"拍题+录音讨论转写"上有没有更顺手的替代？

## 带着这两个用途去查（结论要明确回答）
- **A（最实在）· 现在能否直接当第 0 阶段人肉回路的采集工具**：我们正让她拍错题、舅舅引导口述卡点。Get笔记这类能否**零开发直接拿来**录音+转写、把素材攒下来？这是当下最有用的产出。
- **B · 借鉴其 capture 体验**：将来做归因轮 + 火山方舟 ASR 时，"拍+追加录音+自动提要点"的低摩擦采集值得参考。但**我们不做笔记产品，只借采集环节。**

## 别被带偏（守我们的差异）
- 它"喂教材学知识点地图"是临时手段；我们已有结构化持久知识图谱，不用学这个。
- 它"AI 实时举一反三出题"——我们走**人工过审配题**（质量>数量，P2），不抄实时生成。
- 录音讨论守我们的 **P4 / 共创者 / 追问情绪自适应**（只听和引导、不评判——文章也这么主张，一致）。
