## 一、项目是什么

**一句话**：为用户读高二、明年高考的外甥女（安徽，新高考全国 I 卷，理科）做一个**个性化数学诊断辅导系统**。

**真实痛点**：她数学很弱（150 分卷曾只考 30-40 分）。关键判断是——这不是"高中知识点有漏洞"，
而是**从初中代数运算就开始的系统性断层**。所以目标不是"补全高中数学"，而是**抓基础保底，从 30 分爬到 85-95 分**
（高考数学约 80-90 分是基础题，这是可达的）。

**产品内核（区别于豆包/通用 AI 的地方）**：不是又一个"拍照解题"工具。核心差异是两件通用 AI 做不到的事：
1. **个人化、跨会话持久的知识状态地图**——她每个知识点的掌握情况，结构化、持续更新。
2. **错因归因式追问**——题目做错后不是直接给答案，而是用苏格拉底式追问定位"真正的缺口在哪"，串起断层。

**约束**：用户和外甥女**远程**、用户上班、无法做现场陪伴。外甥女**愿意配合**（这点很重要，是成功的最大前提）。
她**周一到周五在校不能用手机，只有周末回家能用**。系统将来可能分享给她同学。

## 三、已定技术决策（这些是裁决结果，不要轻易推翻，除非有新证据）

### 3.1 系统架构（四层 + AI 管线，叠加在 wrong-notebook 上）
- ① 知识图谱数据层：节点/边/主线/学生状态。
- ② 诊断引擎：初诊(KST-lite) / 错因归因(Newman) / 探针下探。
- ③ 追踪引擎：BKT 掌握度更新 / 知识地图 UI。
- ④ 周流程编排：周末 session 状态机 / 周中纸质包 PDF。
- AI 管线：火山方舟（豆包）全栈优先（VLM 识图 + ASR + 对话 LLM，OpenAI 兼容接口，复用基座 Provider 配置），
  对话备选 DeepSeek。

### 3.2 五条贯穿全局的设计原则（P1-P5，反复引用）
- **P1 图为真相源**：题目照片是唯一可信输入，OCR/VLM 转写只作索引；诊断对话永远同时携带原图。
  （这条直接化解"手写数学识别不准"的最大风险。）
- **P2 结构先于模型**：图谱、边界题、错因规则是显式数据；LLM 只在受约束位置工作，不做底层判定。
- **P3 多问少断**：粗心 vs 概念的结论绝不来自单次答错，必须有"概念题 + 隔天同构变式"双重证据。
- **P4 诊断后台化**：测评对学生弱可见，前台只报增量（地图点亮、新前沿），不报"你又错了"。
  （依据：ALEKS 的周期性 knowledge check 会让弱基础学生体验成"不断被要求证明自己不行"，挫败感极强。
  对一个考 30 分的孩子，默认形态的诊断系统本身就是心理打击。这是被调研反复印证的关键洞见。）
- **P5 周末数字化、周中纸质化**：应用周中输出是一份可打印 PDF（复用基座导出能力），周末是唯一上屏时间。
  （这不是妥协，对一个要戒断刷题焦虑、沉下心的孩子反而更健康。）

### 3.3 知识图谱（M1 里程碑种子数据已就绪，可开工建表）
- **8 条主线**（最终编码，钉死不要改）：M1 数学语言与代数预备 / M2a 函数与指对幂 / M2b 导数 /
  M3 三角与解三角 / M4 数列 / M5 平面向量 / M6 立体几何与空间向量 / M7 解析几何 / M8 计数概率统计。
  另有 M0 地基层（共享根，不是主线，BG001-104，含初中前置）。
- **难度带 tier（A/B/C/D）是比主线更重要的调度维度**：A 托底(Q1-6,Q12-13≈40分) / B 大题第一问(+16~18) /
  C 中档补位 / D 放弃区（压轴，不进她的学习前沿）。**学习前沿排序 = 先按 tier(A>B>C，排除D)，
  同 tier 内按主线高考权重 ROI**。这套贴合"全国I卷大题模块动态轮换、死记题位没用、难度带才稳"的事实。
- 主线真实权重（高考严格年均，供 ROI）：M7解几29.5 > M3三角25.5 > M6立体20.5 > M4数列18.5 >
  M2a函数~16 > M8概统14.5 > M2b导数~10 > M1集合复数10 > M5向量5。
- 边分三类：**prerequisite（必需前置，进图参与下探）/ tool（常用工具，弱依赖边）/
  transfer（思想方法迁移，不进图，降级为节点 teachingNotes）**。
- 误解库：错因标签设计为**两层**（先判错误性质：概念性/程序性/计算性/符号/条件前提缺口/完备性缺口，
  再判知识缺口），并设**跨章复用标签 crossTag**（定义域优先/等价变形守恒/完备分类/取等可达/对象类型意识），
  用于识别"系统性缺口"而非孤立的洞。

### 3.4 输入方案（呼应远程 + 周中无手机）
- 周中三条极简规则：过程写题旁不擦不涂改；做不出的题贴指示标签露页边；卡住处画箭头+几个字写卡点。
  （错因代号①-⑤已废弃——那是系统的活不是她的活。）
- 周末：批量拍照（题目+过程+标记一起拍，整页）→ **题面人工确认 30 秒**（手写过程不确认，靠 VLM 看图）→
  系统分诊挑 2-3 道深诊 → 重做+录音 → Newman 追问 → 探针下探 → 更新地图 → 生成下周纸质包 PDF。
- **不重新编号题目**（照片即题目身份证，VLM 读题面去重）。

## 五点五、诊断方法论与设计理由（核心知识，不要只记结论，要记"为什么"）

> 这一节是前任顾问最看重的部分。前面的决策（tier、P1-P5、schema）是"结论"，这一节是"为什么这么定"。
> 没有这些理由，你只能照搬结论，无法在新情况下做出和前任一致的判断。

### 诊断方法论：中医"四诊"框架（帮用户建立直觉的核心比喻）
诊断不是只看对错，是"望闻问切"四诊合参：
- **望（看解题痕迹）**：不看对错，看草稿纸。她在哪行停笔、哪步开始涂改、式子抄到第几步变形。
  30分的学生草稿纸信息量远大于答题卡。规则：做题不擦不撕草稿，全保留拍照。卡壳位置=病灶位置。
- **闻（听她讲题=费曼检验）**：让她把题讲出来、讲到"为什么这步能这么做"。真懂讲得出因果，
  模式匹配的人讲到一半变成"反正就这么套"。**关键洞见：做对的题也要抽查**——基础差的学生有大量
  "答案对、理解错"的题，这些是知识地图上最危险的暗礁（她自己标记为"会"，实际是空的）。
- **问（结构化追问错题）**：用 Newman 五阶段（阅读→理解→转换→过程→表达）定位最早断点。
  五类错因药方完全不同（知识性→补视频；程序性→带步骤拆解的例题；运算性→指向初中运算自动化缺失，
  每天10分钟限时基础运算；审题转化→训练逐句改写成式子；策略性→按题型归纳"看到什么用什么"）。
  **为什么分类如此重要：推荐视频只能治第1类（知识性）。不归因就推视频，她看了还不会，挫败感加倍——
  这正是市面上"AI推题推课"产品对差生无效的根本原因（它们默认所有错误都是知识性的）。**
- **切（探针题二分下探）**：摸底绝不用整套卷子（对她是羞辱、信息密度低）。沿依赖链二分下探：
  她导数题错→不在导数层反复测，直接出纯"解一元二次不等式"探针；错了再往下探因式分解；
  **直到探到她做得对、做得快的那一层为止——那才是地基真实位置，补救从那里向上盖**。
  每次诊断五六道精选探针，信息量胜过一整张卷子。

### 几个关键的"为什么"（被结论掩盖、但必须传承的设计理由）
- **错因两层标签的精髓**：先判错误性质（概念性/程序性/计算性/符号表示/条件前提缺口/完备性缺口），
  再判知识缺口。**为什么必须分两层：否则系统极易把"概念没懂"误判成"算错了"**——这正是 P3 双证据要防的。
- **crossTag 跨章复用标签的杀手锏用法**：定义域优先/等价变形守恒/完备分类/取等可达/对象类型意识，
  这些底层缺口跨章反复出现。**真正威力在于横切识别**：当她在对数定义域、根式定义域、奇偶性定义域
  上"都"漏，系统应识别为"一个'定义域优先'意识的系统性缺失"，而非三个孤立的洞。这比知识图谱的
  树状结构更能抓到真问题。
- **Eedi 式初诊（C阶段高级设计，别忘了）**：把初诊选择题的"每个错误选项绑定一个特定误解标签"，
  她一选错，系统立刻知道她踩的是哪个坑，无需追问即得归因。误解库种子（错题库调研那20多条
  四联体：错误表现→误解→底层缺口→诊断追问）就是这套的弹药。MaE 数据集（MIT许可）的 JSON
  字段可直接做误解库的 schema 模板。
- **重做+录音为什么必须"重做"而非"回忆"**：到周末她已不记得周二那道题"当时为什么卡"，且事后回忆
  会被污染（她会编一个听起来合理的理由）。让她当场重做，思维过程被现场重新生成——这是 think-aloud
  文献"并发口述优于事后回忆"的应用。重做顺利+周中标记"算错"→判熟练度问题；重做又卡同一步→活的卡点。
- **"知识地图即治疗"的心理学设计**：考30分的孩子内心叙事是"我数学不行"（针对身份的、不可解的全局判断）。
  地图把它转译成"你有23个具体的洞，其中7个是初中的，先补这7个"（可数、可执行的清单）。
  **从"我不行"到"我还差23个"的叙事转换，本身就是治疗的一半**。这张地图是产品的灵魂界面，比对话窗口重要。
  每补掉一个点，地图上一块从红变绿。

### 工程上的关键抓手
- **黄金测试集**：开发第一周，让她用真实教辅、真实手写、真实家里灯光拍 20 张照片建回归集。
  每次换模型/改 prompt 都跑一遍人工评分（题面识别须100%，手写过程理解抽查）。
  **模型选型不看评测榜单，看它在她的字迹和她那几本教辅版式上的真实表现**。这20张也是第0阶段第一批素材。
- **输入双通道，不赌纯OCR**：拍照 + 口述"哪里卡住"两条通道。手写数学OCR至今全球无完美方案
  （Wrong Notebook 的真实卡脖子点就是拍照识题超时、图片传不上、复杂解析失败，不是模型不够聪明）。
- **真题图片格式的预警价值**：用户的真题docx公式全是图片、纯文本抽取不可行——这恰好预示了
  她拍教辅照片时VLM会遇到的同款难题。处理真题=提前给识别管线做真实压力测试。

## 六、与用户协作的方式（重要，决定你帮不帮得上）

- 用户的标准工作流是：**外部调研/项目AI产出 → 你评审裁决 → 确认后落地**。他很依赖你做"质量闸门"。
- 评审时要敢于**抓 AI 产出的典型病**：擅改约定编码（3-2 干过）、公式存成图片导致无法入库（18万字那份）、
  伪造链接/题目（B站、真题）、把教师经验冒充实证。每次接外部材料，先验真实性再谈采纳。
- 用户不懂代码，所以涉及技术要用**大白话+类比**，并主动替他考虑"安全护栏"（误删、密钥泄露、合并冲突、卡在第一步）。
