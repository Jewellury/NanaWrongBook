# 竞品速评 · 术子错题宝

> 来源：用户提供的《术子错题宝·完整使用手册》PDF（19 页，纯图片，2026-06 渲染逐页阅读）。
> 出品方：术子智能信息技术（重庆）有限责任公司。微信小程序。

## 它是什么
9 学科（数/语/英/物/化/生/史/地/政）**通用 AI 错题本**。完整功能：
- OCR 拍照录入错题（分别填「我的答案」+「正确答案」+ 可选「来源」如"期中/练习册P23"）
- 错题库：多维检索（关键词/掌握度/科目/日期）、卡片（科目色 + 掌握徽章 + 知识点≤3 + 预览）
- AI 智能解析：**单次**给"错因分析 + 正确解法 + 关联知识点"
- AI 生成同类题 3-5 道（举一反三）+ 测验模式
- 掌握度三级（陌生/熟悉/已掌握）+ 艾宾浩斯式间隔复习 + "今日待复习"智能推荐
- 学习报告（薄弱点 % 维度）、学习计划打卡、提分专卷 PDF 导出（选题型+数量→A4）
- 等级系统：学童→书生→秀才→举人→进士（按掌握率 0/10/30/50/75%）

## 战略结论：它是「市场验证 + 反面教材」，不是要追赶的对手
- 通用 AI 错题本这条路，它做得又全又精——**我们拼功能广度必输，也不该挤进去**。
- 但它恰恰**没有**我们的两个内核（handoff 差异点）：
  1. **没有连起来、带前置依赖的知识图谱诊断**。它的"掌握度"是孤立错题/知识点标签，不会"导数错→沿依赖链下探到初中因式分解"，抓不到**系统性断层**。
  2. **没有错因归因式多轮追问 + 双证据**。它的 AI 错因是**单次结论式**解析（"你混淆了概念A和B"），默认所有错误都是知识性的→直接推同类题——**正是 handoff 说的对差生无效的模式**。
- → 印证我们的定位正确且是市场空白：为**一个深度用户做诊断深度**，而非功能广度。

## 值得借鉴（战术层，不动战略）
1. **★记录"她的实际错误答案"，不只对错**。错误答案是归因金矿（她写了什么 ≫ 错没错的信息量）。我们 `ProbeRecord` 现在只有 `correct` 布尔，建议未来加 `studentAnswer` 字段，喂 P3/Newman/复诊。→ 记入下一轮（归因）。
2. **来源标注**（期中/练习册页码）——轻量回溯，录入时可加。
3. **间隔复习调度**（"今日待复习"）——与我们纸质包选题结合：周中练 = 到期该复诊的 + 学习前沿。我们有 BKT（比它的三级更细）但还没有调度层。→ 并入运营回路/复诊。
4. **报告的"维度 %"呈现**——并入运营回路 backlog，但守 **P4 双层**（给她的正向 / 给舅舅的诊断）。

## 明确别照抄
- **等级系统绑掌握率 %**：对一个被数学打击的孩子 + 共创者框架，这是另一种"证明你不行"的秤（P4 / ALEKS 挫败感警示）。要成就感可以，**绝不绑掌握率当门槛**。
- **9 学科广度 / 实时 AI 生成同类题 / 重度纯 OCR 依赖**：分别与我们的 MVP 敢砍、配题人工过审（P2 结构先于模型）、图为真相源（P1 不赌纯 OCR）相悖。

> 一句话：术子证明了"通用错题本"市场成熟且拥挤，也反衬出"系统性断层诊断 + 归因追问"是真空白。继续走我们的窄而深，别被它的功能清单带跑。
