# 工单 · 外部调研 AI：错题采集到诊断闭环案例调研

> 这份工单给外部调研 AI。它没有 Nana 项目上下文，所以本文已经包含最小背景。
>
> 你的任务不是写产品方案，也不是做技术选型，而是去找可借鉴的产品案例、研究案例、开源项目、交互模式和教育方法。重点回答：我们设想的“拍错题 -> 即时反馈 -> 批量诊断 -> 复习回流”有没有类似实践，哪些值得借，哪些不要学。

## 项目最小背景

Nana 是一个面向数学基础薄弱高中生的错题收集与个性化诊断辅导系统。

目标用户是一名正在准备高三复习的高中生。她现在高二，暑假和高三期间会积累大量数学错题。我们希望系统不是单纯“保存错题”，而是用这些错题帮助她诊断知识薄弱点、解题方法薄弱点、错因模式，并生成后续练习和复习计划。

我们计划做浏览器应用，不做 App，不做微信小程序。

## 我们设想的核心使用流程

### 阶段 1：前期大量收集错题

用户打开浏览器应用后：

1. 点击拍照。
2. 对题图做旋转、裁剪、涂抹或遮罩，聚焦到本题，提升后续识别准确率。
3. 语音补充自己当时怎么想、哪里卡住、哪里不懂。
4. 系统听完后立刻给一个“初步诊断反馈”，而不是只说“保存成功”。
5. 系统提醒她继续拍下一题。

我们希望单题反馈像一个可信任的人在回应她：

- “我先把这道题收好了。”
- “你刚才提到卡在某一步，这可能和某个知识点或某类方法有关。”
- “这只是初步线索，不是最终诊断。”
- “再拍几道类似题后，我可以一起看是不是同一类问题。”

### 阶段 2：一批错题拍完后的综合诊断

当她一次性拍完一批错题后，系统生成综合诊断报告：

- 这些错题主要落在哪些知识点。
- 哪些知识点或知识链条薄弱。
- 哪些数学方法族薄弱，比如分类讨论、数形结合、定义域优先、换元、构造函数、配方法、转化与化归等。
- 主要错因模式是什么，比如读题偏差、概念不清、方法选错、过程断裂、计算失误、表达或检查不足。
- 推荐从知识地图或方法地图的哪个位置开始补。
- 推荐相关讲解视频。
- 给她出一组针对性的练习题，可以打印出来写。
- 不会的再拍回来，继续进入诊断循环。

### 阶段 3：错题基本收集齐后的回顾复习

后期新增拍题会减少，她打开系统主要是复习：

- 通过知识地图和数学方法地图看自己经常错的地方。
- 点进某个节点，看到之前做错的题、当时的诊断、后续补充。
- 每个错题 case 可以继续补充新理解。
- 系统根据她后续练习的表现更新掌握度。
- 系统提醒她该复习哪些错题、做哪些变式题。
- 题目来源可以是她拍过的错题，也可以是相似题或变式题。

## 本轮调研要找什么

请围绕下面 8 个方向找可借鉴案例。优先找真实产品、开源项目、论文、教育实践案例。中文、英文都可以。

### 1. 拍题 / 上传题图后的即时反馈

找有没有产品或研究做过：

- 学生上传错题后，不只是保存，而是立刻给初步诊断。
- 反馈不是“判分”，而是“我看到你可能卡在哪里”。
- 反馈带有不确定性表达，比如“可能”“初步线索”“还需要确认”。

请重点看错题本、AI 家教、拍照搜题、学习诊断、作业批改类产品。

### 2. 单题 case 的持续补充机制

找有没有系统把一道错题当成长期 case：

- 题图、语音、转写、摘要、标签、诊断、后续补充、复练记录都挂在同一个 case 上。
- 学生后来可以回来补充“我现在懂了什么”。
- case 会随着练习和复习持续更新。

### 3. 批量错题后的综合诊断报告

找有没有产品或研究支持：

- 一次上传多道错题。
- 系统按知识点、题型、方法、错因做聚类。
- 给出一份面向学生或家长的诊断报告。
- 报告不是冷冰冰的数据表，而是能指导下一步复习。

### 4. 知识地图 + 方法族地图

我们不只关心知识点，也关心数学方法。

请找类似实践：

- 知识图谱用于错题诊断。
- 方法图谱 / 解题策略图谱 / skill map / strategy map。
- 把错题归因到“知识点”和“解题方法”两个维度。
- 方法族示例：分类讨论、数形结合、换元、构造、配方、归纳、反证、转化、定义域优先、参数分离等。

重点回答：有没有成熟产品或研究同时做 knowledge map 和 method / strategy map？

### 5. Newman 归因或错误类型诊断

我们正在考虑是否提前做 Newman-lite 归因。

请调研：

- Newman Error Analysis 在数学错题诊断中的应用案例。
- 是否有产品把错误分为读题、理解、转化、过程、计算、表达等类型。
- AI 家教或智能辅导系统如何做“错因诊断”。
- 哪些错误类型适合首期做轻量判断，哪些必须靠追问或人工确认。

### 6. 推荐视频与知识节点 / 方法节点挂接

我们希望根据诊断结果推荐视频。

请找：

- 学习系统如何把讲解视频绑定到知识点。
- 是否有做视频切片、章节定位、时间戳推荐的案例。
- 是否有按“方法”而不是只按“知识点”推荐视频的案例。
- 首期是否可以先人工精选视频链接 + 起止时间，后续再做自动切片。

### 7. 练习题、相似题、变式题生成

我们希望系统根据错题出题：

- 可以从学生拍过的错题里复练。
- 可以推荐相似题。
- 可以做变式题，测试是否真的掌握。

请调研：

- 错题本产品如何做相似题推荐。
- 变式题生成有哪些靠谱做法。
- AI 生成数学题有哪些风险。
- 是否有“模板化变式”或“人工审核题库 + AI 推荐”的实践。

### 8. 掌握度更新与复习排程

请找：

- 错题复习如何更新掌握度。
- BKT、IRT、FSRS、SM-2 在错题复习中的实践。
- 系统如何利用“做题结果反馈”更新知识点状态。
- 怎样避免学生只拍题不反馈，导致掌握度无法更新。

## 你要特别注意的边界

请不要只列产品名。每个案例都要回答“它对 Nana 有什么启发”。

同时请注意：

- Nana 的首期不能变成大而全系统。
- 浏览器应用是前提，不要推荐必须原生 App 才能实现的方案，除非只是借交互思路。
- 用户是基础薄弱高中生，低压感很重要，不能把界面做成诊断仪表盘或惩罚系统。
- 诊断反馈要有帮助，但不能假装系统百分百确定。
- 数学题生成风险很高，任何“AI 自动出题”都要说明可靠性和审核问题。

- 报告能帮助我们判断“采集壳是否应该提前加入即时诊断反馈”。

- 报告能帮助我们设计“批量错题综合诊断报告”。
- 报告能区分“知识点缺口”和“方法族缺口”。
- 报告不会把首期 MVP 扩成不可落地的大系统。
