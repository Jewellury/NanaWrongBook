# 工单 · 外部调研 AI：高中数学讲解视频资源补充调研

> 这份工单给外部调研 AI。它没有 Nana 项目上下文，所以本文包含最小背景、已有材料、不要重复做的部分，以及本轮需要增量补充的部分。
>
> 本轮目标不是“再列一堆 B 站老师”，而是把讲解视频变成 Nana 诊断系统可用的“处方资源”：能挂到知识点、方法族、错因类型，并且看完后能复诊验证。

## 1. 项目最小背景

Nana 是一个面向数学基础薄弱高中生的错题收集与个性化诊断辅导系统。

目标学生当前数学基础较弱，后续会在暑假和高三期间持续收集错题、做诊断、补薄弱点。系统的定位不是自建讲课平台，也不是无限聊天讲题，而是：

1. 收集错题和学生当时的思路。
2. 诊断错题落在哪些知识点、数学方法、错因模式。
3. 给出下一步复习建议。
4. 推荐合适的外部讲解视频或资料。
5. 看完后用题目复诊，确认是否真的补上。

请特别注意：视频推荐不是终点。它类似“医生开药”，后面必须有“复诊验证”。如果学生看了视频仍然不会，系统应能判断是视频不合适、知识根源更深，还是需要换一种讲法。

## 3. 第一轮结论，默认先认可

以下结论本轮默认认可，除非你找到强证据需要修正：

1. 弱基础学生不适合一上来使用“压轴题技巧课”“秒杀课”“模拟卷讲评课”。
2. “数学微课帮基础 A”可作为很多高中概念的主线补课资源。
3. “夏老师的数学课”适合初高中衔接和运算断层。
4. “乐乐课堂”适合 2-6 分钟预热或复盘，不适合单独承担系统主线。
5. “一数”适合第二阶段框架补全，不适合纯零起点第一棒。
6. “圆脸”适合函数、向量等抽象概念的二次解释。
7. 分式基本运算等初中前置薄弱点，第一轮没有找到足够稳的主推视频，不要硬凑。

本轮不要只复述这些结论，要做增量。

## 4. 本轮调研问题

### 4.1 核验已有资源是否仍可用

请核验第一轮报告中的主要视频资源：

- 视频或合集是否仍存在。
- 标题是否变化。
- 搜索主键是否仍稳定。
- 是否有更稳定的官方合集页、合集编号、BV 号或播放列表链接。
- 是否存在搬运、侵权、重复上传、标题党等风险。

输出时请给出可点击链接、BV 号或更稳定的定位方式。不要只写“搜某某老师”。

### 4.2 补齐薄弱知识点资源

请围绕弱基础高中生，补充第一轮缺口，尤其是：

- 分式基本运算。
- 一元一次 / 二次方程基础。
- 一元一次 / 二次不等式基础。
- 绝对值、根式、因式分解。
- 函数定义域、值域、单调性、奇偶性。
- 指数、对数的最底层概念。
- 三角函数弧度制、诱导公式、图像。
- 平面向量基本概念、坐标化、数量积。
- 概率统计中事件、古典概型、均值方差。

如果某个点找不到合适 B 站视频，可以推荐教材章节、讲义、图文资料、可打印资料，或者明确写“暂缺，不建议硬推视频”。

### 4.3 从“知识点推荐”扩展到“数学方法族推荐”

Nana 后续可能不只有知识地图，还会有“数学方法族地图”。请调研视频资源是否能按方法族挂接，例如：

- 分类讨论。
- 数形结合。
- 定义域优先。
- 换元。
- 配方法。
- 构造函数。
- 转化与化归。
- 建系 / 坐标法。
- 参数思想。
- 函数与方程思想。

请区分两类视频：

- 适合弱基础学生的“方法入门课”。
- 适合学生已有基础后的“题型框架课”。

不要把高难压轴方法课误推荐给当前学生。

### 4.4 从“视频整集”细化到“视频切片”

请判断是否需要做视频切片或时间戳推荐。

重点回答：

- 哪些视频整集就适合推荐。
- 哪些视频太长，需要只推荐某个时间段。
- 是否能从 B 站章节、评论区课代表、字幕、合集目录中获得稳定时间戳。
- 人工维护视频切片的最小字段应该是什么。

建议输出一个字段模板，例如：

```ts
{
  title: string;
  source: "bilibili" | "textbook" | "handout" | "other";
  url?: string;
  bv?: string;
  searchKey: string;
  uper?: string;
  duration?: string;
  startTime?: string;
  endTime?: string;
  role: "预热" | "主线" | "二次解释" | "方法课" | "复盘";
  level: "零基础" | "基础薄弱" | "已学过一遍" | "提高";
  attachTo: "knowledgeNode" | "methodNode" | "errorType";
  attachKey: string;
  reason: string;
  cautions?: string;
}
```

可以修改这个模板，但请说明理由。

### 4.5 建立“诊断结果 -> 视频处方”的推荐规则

请不要只给资源表，还要给推荐规则。

例如：

- 如果诊断是“知识点没概念”，应该推荐什么类型的视频。
- 如果诊断是“方法不会选”，应该推荐什么类型的视频。
- 如果诊断是“运算断层”，是否应该回初中前置视频或讲义。
- 如果诊断是“读题偏差 / 表达不完整 / 计算粗心”，是否不该推荐视频，而该推荐练习或检查清单。

请特别说明：哪些错因不适合靠视频解决。

### 4.6 设计“看完视频后的复诊验证”

请调研或提出可借鉴方案：

- 看完视频后，系统应该如何确认学生真的理解了。
- 复诊题应该是原错题、同构题、边界题、还是更基础的探针题。
- 什么时候把知识地图节点从红 / 黄变绿。
- 如果复诊失败，系统应该换视频、下探前置知识，还是换成讲义 / 人工辅导。

这部分不需要写代码，但要给产品和数据设计建议。

### 4.7 版权与可持续性风险

请补充：

- B 站视频能否直接内嵌，还是只应该外链。
- 是否存在视频下架、改名、合集重排的风险。
- 如何保存最小可恢复信息。
- 是否应该优先推荐官方 / 原创作者内容，避免搬运。
- 如果未来产品化，视频推荐是否有版权、商业合作或合规风险。

## 5. 输出格式要求

请按以下结构输出调研报告：

### A. 一页结论

用 5-10 条 bullet 总结：

- 第一轮结论哪些保留。
- 哪些需要修正。
- 最大缺口是什么。
- Nana 首期应该怎么用视频资源，不应该怎么用。

### B. 已有资源核验表

字段建议：

| 原推荐 | 当前可用性 | 稳定链接/BV号 | 是否建议保留 | 风险 | 备注 |

### C. 增量资源表

字段建议：

| 知识点/方法族/错因 | 推荐资源 | 链接/BV号 | 适合程度 | 推荐角色 | 时长/切片 | 推荐理由 | 风险 |

### D. 方法族视频表

单独列出可挂到方法族地图的资源，不要混在知识点表里。

### E. 推荐规则

输出“诊断类型 -> 推荐动作”的规则表。

至少覆盖：

- 知识性 gap。
- 方法性 gap。
- 运算断层。
- 读题偏差。
- 过程断裂。
- 计算失误。
- 表达 / 书写 / 检查不足。

### F. 复诊闭环建议

说明看完视频后，应该如何复测、如何记录结果、如何更新掌握度。

### G. 最小落地版本建议

请给 Nana 一个务实建议：

- 首期只维护多少条视频资源比较合适。
- 优先覆盖哪些节点。
- 是否需要做时间戳切片。
- 人工维护工作量大概是多少。
- 哪些内容应明确暂缓。

## 6. 边界与禁区

1. 不要把 Nana 变成“视频课平台”。
2. 不要推荐压轴题技巧课作为弱基础学生第一入口。
3. 不要默认“看视频 = 会了”。
4. 不要把所有错因都导向视频。
5. 不要用不存在或不稳定的链接糊弄，找不到就写暂缺。
6. 不要只按知识点推荐，也要考虑方法族和错因类型。
7. 不需要写代码，不需要设计数据库，但可以建议字段。

## 7. 验收标准

这份调研报告应能帮助我们回答：

1. 现有 B 站视频清单哪些可以直接保留。
2. 哪些关键薄弱点还缺可靠资源。
3. 视频如何挂到知识图谱节点、方法族节点、错因类型。
4. 诊断后何时该推荐视频，何时不该推荐视频。
5. 看完视频后如何复诊验证，避免假性闭合。
6. 首期最小可落地的视频资源库应该长什么样。
