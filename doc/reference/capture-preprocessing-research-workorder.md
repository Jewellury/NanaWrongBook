# 工单 · 采集预处理（图像 / 音频）调研

> 目标：查清"拍照识别 + 语音识别"前该有哪些**预处理**、哪些现成可借、对我们的**最小必要集**是什么。
> 性质：**调研，不是现在建**。真正必要性待她真实照片/录音来验。

## 给调研 AI 定调（场景与已知）
- 两路输入：① 手机拍教辅错题（含手写过程 + 题面）→ 火山方舟豆包 **Pro VLM** 识图；② 周末口述卡点录音 → ASR 转写。
- 已知卡脖子（来自项目史）：wrong-notebook 的真实痛点是**拍照识题超时 / 图片传不上 / 复杂解析失败**，不是模型不聪明。
- 用户画像：青春期孩子（**对使用摩擦极敏感**）+ 远程异步 + 家庭手机、可能网络差。
- **强 VLM 前提**：豆包 Pro 对清晰图鲁棒性已不错（150DPI 真题直接识别很准）。所以**核心判断 = 对她真实手机拍的照片，哪些预处理"必要" vs 传统 OCR 遗留的"多余"**。别用老 OCR 思路套强 VLM。

## 必须守的原则：P1 图/音为真相源（写进调研框架）
- **原图、原音无损保留 = 真相源**；预处理只产"识别友好的派生版"喂模型。
- 增强可能**有损**（降噪削掉"约分/通分"等关键词、二值化丢浅色笔迹、过度锐化造伪影）。
- 因此：**每一个有损手段都要标注"会不会改变真相"**，真相判定永远回原始版。

## 用这个三层框架组织调研（从便宜到贵，重点查中间两层）
**源头拍录好（已有拍照说明）** ＞ **采集即时质检 + 引导重拍/重录** ＞ **自动校正/增强** ＞ 送识别。

## 一、图像预处理 要查
- **质检拦截**：模糊/抖动检测、过暗/过曝、分辨率不足、倾斜/旋转检测 → 当场提示重拍。
- **几何校正**：透视矫正（deskew/dewarp，把斜拍的纸拉正）、自动找纸张边界裁边、旋转纠正、EXIF 方向。
- **增强**：去阴影/去反光、对比/亮度、二值化、去噪。
- **传输**：在不伤识别质量前提下压缩、分辨率/格式标准化（应对"图传不上"）。
- **端上即时 vs 服务端**：哪些适合手机端当场做（拍糊立即重拍，最贴 #1 风险）。
- **现成方案**：OpenCV、开源/商用文档扫描 SDK、手机端 document-scanner 组件、云厂商文档识别自带的预处理、VLM 厂商的**输入规范建议**。
- **强 VLM 下哪些可省**：豆包/主流多模态模型对倾斜/阴影/低光的容忍度，哪些传统预处理已无必要。

## 二、音频预处理 要查
- **质检**：太吵/太小声/无人声检测 → 提示重录。
- **流程**：降噪/去回声、增益归一化（音量忽大忽小）、VAD 静音裁剪、**说话人分离**（舅舅 + 她两个声音）、分段、采样率/格式标准化。
- **现成方案**：各 ASR 厂商（火山方舟/讯飞/通义听悟）**自带哪些预处理**（很多自带降噪/VAD/diarization，可能不用我们做）、开源库（WebRTC VAD、RNNoise 等）。
- **守 P1**：标注哪些有损（降噪可能削关键词）。

## 输出要求
1. **图像、音频各一张表**：手段 / 作用 / 端上 or 服务端 / 现成方案 / **是否有损（改不改变真相）**。
2. **对我们场景的"最小必要集"建议**：青春期孩子摩擦敏感 + 远程 + 强 VLM 前提下——**哪些现在就值得做、哪些等真实数据暴露问题再上、哪些可省**。
3. **即时质检引导重拍/重录**的产品做法举例 2-3 个（谁做得好、怎么做的）。
4. 诚实标来源与置信度，查不到就说查不到，不编。

## 别被带偏
- 我们不是要建一个重型预处理服务；优先"源头引导 + 端上即时质检 + 复用厂商/库现成能力"。
- 强 VLM 时代，宁可少做预处理、先看真实照片 VLM 扛不扛得住，再按暴露的问题补——别预先过度工程。
