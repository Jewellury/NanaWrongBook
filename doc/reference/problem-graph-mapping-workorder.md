# 工单 · 错题 ↔ 知识图谱 关联模型 + 批次上图交互（出方案）

> 产出：一份**设计方案（蓝图）**——错题怎么挂到知识图谱、怎么"对着地图看本批次新题"。
> **不写实现代码、不建界面。** 建设待用户拍板。
> 与 `frontend-architecture-workorder.md` 的命名空间/复用规约一致；这份专攻"题↔图谱关联 + 批次上图交互"。
> 产出后交 **Codex 架构评审 + 参谋长产品/原则评审**。

## 立意（先读，定方向）
题目管理的使命**不是"给题编号方便对题"**，而是**"让每道错题接到知识地图的节点上，远程讨论时对着地图看这批题落在哪些知识点"**。身份/引用是次要的，**挂节点是主干**。

## 一、数据模型（case 为单位，挂节点，复用既有表）
- **单位 = 一道错题 = 一个诊断 case**，不是"一张照片"。一个 case 可挂多张照片（附件、只取目标题）；一张照片可能含目标题 + 夹带邻题（只认有手写的目标题，见 handheld 分析工单）。
- **复用既有表，别另起孤立相册**：
  - 错题本体落在哪？给出权衡并定：复用上游 **ErrorItem**（顺带白嫖它的录入/标签/导出 PDF），还是新建我方 case 表挂接 ErrorItem。
  - **节点关联复用 M1 既有的 `MistakeNode`（mistakeId ↔ nodeId, confirmed）**——VLM 给候选节点 → 人工确认。**一道综合题可挂多个节点**（定义域 + 单调性），MistakeNode 本就多对多。
  - **批次**概念（周末一次上传）：新增一张 batch 挂接表（不改上游）。
  - 照片附件 / VLM 原始输出 / 提取题面（供搜索）：新表挂接。
- **守**：不改 wrong-notebook 上游表；新功能新表挂接。

## 二、节点映射流程（VLM 候选 → 人工确认，守 P2）
- VLM 识题已能输出 `knowledge_points` 自然语言关键词（见 handheld-report）。要把它**映射到我们的 nodeId**（M1-xx/BG-xx）。
- **别让 VLM 直接定 nodeId**（它不懂我们的编码本，会瞎标，违 P2 结构先于模型）。设计成：**关键词 → 候选节点（用图谱节点名做相似匹配给候选）→ 人工（舅舅/她）确认**（写入 MistakeNode.confirmed）。
- **处理"题考到图谱里还没有的节点"**：当前节点库只有 batch1（M1/M2a A 层 + 地基），真实错题会落到 M3–M8 等**还没建的节点**。方案要给"未覆盖节点"的标记机制——**这反过来驱动配题/建图长尾**（哪些域该补，接回运营回路 backlog）。这是这套设计最有机的一环，务必包含。

## 三、批次上图交互（核心，重点设计概念）
- 流程：周末上传一批 → 每道题给候选节点 → **在知识地图上高亮"本批次新题落在的节点"** → 舅舅和她**对着地图讨论"这批错题集中在哪几个点"**。
- 把"对题号讨论"升级成"对着地图看知识分布"。
- **守 P4（渐进展开 + 正向）**：地图上呈现"这批点亮/触达了哪些待攻克点"，**不是"你又错了一堆"**；措辞、配色正向，不一次铺一片红。
- **顺带解决人-系统互指**：她说"那道复数题"→ 地图复数节点高亮 + 其下挂的本批错题。**引用天然锚在节点，不靠题号。**
- 交互要素（给概念 + 线框级即可，不实现）：批次视图、地图上批次高亮、点节点看其下本批 case 卡片（题图 + VLM 题面 + 候选节点 + 确认/改节点入口）。
- **守前端不冲突**：自己的路由命名空间 + 自己的组件，复用上游只 import 不改（见 frontend-architecture-workorder.md）。

## 四、去重
同题被重复拍（她重拍 / 真题在两本书都出现）：**题面归一化 + 节点** 做去重候选 → 提示"这题好像拍过"→ 人工确认是否合并。方案给策略，别留空。

## 五、范围与守则
- **不做**：实现代码、不建全部界面、不引重框架。
- **复用**：ErrorItem / MistakeNode / KnowledgeNode / StudentNodeState / 图谱。
- **守**：不改上游、前端命名空间不冲突、P1（原图真相源）/P4（地图正向）/P5（周末批量）。
- 测试集那张 `test-01→题号+节点` 映射表是**另一件小事**（trivial，单独做），不在本方案范围。
